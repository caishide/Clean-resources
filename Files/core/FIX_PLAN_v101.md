# V10.1 修复计划（根据评审缺口）

- 退款/调整闭环：补齐 AdjustmentBatch/Entry 模型，退款前/后 Finalize 分支、负向流水同步余额/PV/积分（已落地）；后续需接入后台审核入口与批量执行。
- 待处理奖金释放：自动释放已补 balance+流水；后台手工释放需校验 release_mode、展示待处理列表、加入审核结果/备注。
- 幂等与防重：直推奖幂等（已落地）；层碰/其他奖金需补防重校验与重放防护。
- PV 扣减与对账：周结扣减按左右分别记录（已落地）；补报表导出（周/季度/待处理/PV/莲子），并对齐数据字典字段。
- 分布式锁与调度：结算锁改为 Cache lock，新增季度结算命令与调度（已落地）；需在后台加手动触发按钮、执行记录展示。
- 数据模型补充：user_assets 表/调整引用字段（如 user_points_log.adjustment_batch_id）按数据字典补全，代码读写同步。
- 负余额风控：提现/转账需阻断 balance<0 场景（待补口径）；钱包流水 post_balance 全量回填。
- 报表与审核界面：后台待处理奖金审核、调整批次列表/审核、结算报表导出页面、季度分红达标/未达标清单等界面待实现。
- 测试覆盖：针对幂等、退款冲正、周/季度结算、负余额、手工审核补充 PHPUnit/集成用例，压测脚本按计划准备。
