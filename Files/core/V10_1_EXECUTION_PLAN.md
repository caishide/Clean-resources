# V10.1 逐步实施任务计划（建议）

## 目标
补齐用户/后台需求文档中的缺口，并建立可验证的结算/对账闭环。

## 阶段 1：制度配置与核心口径（1-2 天）
- 迁移 `bonus_configs`，初始化首个版本（从现有配置落库）。
- 后台配置页支持版本切换、激活与审计记录。
- 统一 PV 口径：周/季结算使用 `pv_ledger` 统计。
- 验证：手动创建 1 条版本并激活，检查 `config_snapshot` 写入。

## 阶段 2：退款与冲正链路完善（1-2 天）
- 发货后退款入口：后台订单支持退款并触发调整批次。
- 冲正逻辑：PV/奖金/莲子 A/B 写负向流水，待处理奖金同步处理。
- 审计：自动/人工 finalize 均落 `audit_logs`。
- 验证：构造已发货订单，执行退款并检查台账与余额变化。

## 阶段 3：用户端关键页面补齐（3-5 天）
- 奖金中心（直推/层碰/对碰/管理/季度分红/待处理）。
- 周结算列表/详情、季度分红页面。
- 莲子积分页面与签到入口。
- 验证：与 API 对齐字段，补齐空态与解释文案。

## 阶段 4：后台结算与报表（2-3 天）
- 周结算/季度结算管理页面：预演/执行/锁提示。
- 报表导出：周结算/季度分红/待处理奖金/PV/莲子。
- 风控指标：K 值曲线、拨出率、待处理余额汇总。

## 阶段 5：测试与验收（持续）
- 单元测试：K 值计算、封顶、幂等、负余额提现。
- 集成测试：发货→结算→退款冲正全链路。
- 数据核对：订单、PV 台账、transactions 三表闭环。
